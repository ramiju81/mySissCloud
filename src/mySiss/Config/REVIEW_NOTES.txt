Revisión rápida del SQL original y correcciones aplicadas para IRIS for Health:

1. Tipos no válidos o poco seguros para IRIS:
- string -> VARCHAR(...)
- "String" -> VARCHAR(...)
- hash -> VARCHAR(255)
- datetime -> TIMESTAMP
- bigdecimal -> NUMERIC(18,2)
- AñoMes -> AnioMes

2. Relaciones corregidas:
- WS_Asegurador.ID_Asegur ahora referencia MS_Asegur.ID_Asegur.
- Se eliminó la FK WS_Asegurador.ID_Asegur -> WS_Paciente.ID_Asegu porque mezclaba catálogos con transaccional.
- HO_Preliquida.ID_Periodo ahora referencia HO_Periodo.ID_Periodo.
- AU_ListasPosi quedó con PK compuesta (CodigLista, CodigRequi).
- AU_Validacion y AU_PosiHallaz se ajustaron para usar la clave compuesta de AU_ListasPosi.
- AU_CabeHallaz.NumeroCaso quedó BIGINT para poder referenciar WS_Casos.ID_Caso.

3. Relaciones ambiguas dejadas sin amarre fuerte:
- MS_ClienteMod.id_EstadoActual
- AU_CabeHallaz.SedesAudit
- AU_CMPosicion.NotaAudito
- AU_CCPoscHall.EvidAudito

- mySiss/config/schema_mysiss_iris.sql
- mySiss/config/SchemaNative.cls
